# Restart Agents to Make Changes Active

With every change to a Flow (by clicking Save), you'll have a new Flow version. In order for those changes to take effect within your Agent (which puts Flows live by applying compute resources), you need to restart the agent and ensure you are selecting the latest Flow version.

If you have a new version of a Flow available, it will show 'Update Available' in your Flow Version column for your Agent.

<figure><img src="/files/ARCrqs5c9ClgmVqphbiu" alt=""><figcaption><p>Agent List - New Flow Version Indicator</p></figcaption></figure>

On Agent restart, if there is a new Flow available you will also be prompted to select it.

<figure><img src="/files/q4z3xwWR9PP78LGyA32E" alt="" width="329"><figcaption><p>Agent Restart - New Flow Version Prompt</p></figcaption></figure>

If you need to revert to a previous Flow version simply use the Version Control feature, restore the Flow version you want to use (which will create a new / updated version number), and run that with your Agent.
